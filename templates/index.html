The Forge — From Scratch
Model Architecture

A decoder-only transformer language model following the GPT-2 architecture (Radford et al., 2019), implemented entirely from scratch in PyTorch. The model uses pre-norm residual connections (Xiong et al., 2020) and learned positional embeddings. All components below were written by hand — no library abstractions.

1. Input Representation

Raw text is fed into the model as a string. Before any neural computation, the text must be converted into a numerical representation the model can process. The input string is first split into subword tokens, then each token is mapped to an integer ID from a fixed vocabulary.

Example: "virtue is" → ["virt", "ue", " is"] → [38834, 518, 318]

2. Byte-Pair Encoding Tokenizer

We use the GPT-2 tokenizer via OpenAI’s tiktoken library. Byte-Pair Encoding (BPE) iteratively merges the most frequent pairs of bytes/characters in a training corpus to build a vocabulary of 50,257 subword tokens. This allows the model to handle any text, including rare words, by decomposing them into known subword units.

A special <|endoftext|> token (ID 50256) is inserted between documents in the training corpus. This teaches the model to recognize document boundaries and prevents it from generating text that bleeds across unrelated passages.

3. Embedding Layer

The embedding layer transforms discrete token IDs into continuous vector representations. Two separate embedding tables are used:

Token Embedding (tok_emb): A lookup table of shape (50257, 384) mapping each token ID to a 384-dimensional learned vector. These vectors encode semantic meaning — tokens with similar meanings will have similar vectors after training.

Positional Embedding (pos_emb): A lookup table of shape (256, 384) encoding each position in the sequence (0 through 255). Unlike the sinusoidal encodings in the original Transformer (Vaswani et al., 2017), these are learned parameters, following GPT-2’s design.

The two embeddings are summed element-wise and passed through a dropout layer (p=0.1 during training) for regularization:

x = Dropout(tok_emb(tokens) + pos_emb(positions))

Output shape: (batch_size, seq_len, 384)

4. Transformer Block (×6)

The core of the model is a stack of 6 identical transformer blocks. Each block contains two sub-layers — a multi-head self-attention mechanism and a position-wise feed-forward network — both wrapped with pre-norm residual connections. The block structure follows the Pre-LN Transformer variant (Xiong et al., 2020), which applies LayerNorm before each sub-layer rather than after, improving training stability.

The complete data flow through one transformer block:

x′ = x + Dropout(MultiHeadAttention(LayerNorm(x)))
x″ = x′ + Dropout(FeedForward(LayerNorm(x′)))

4a. Layer Normalization

Applied before each sub-layer. Normalizes activations across the embedding dimension with learned scale (γ) and shift (β) parameters. This stabilizes the distribution of intermediate activations, preventing gradient explosion or vanishing across the 6-layer stack.

LayerNorm(x) = γ · (x − μ) / √(σ² + ε) + β

384 (last dim)
EPSILON
1e-5
LEARNED PARAMS

VARIANCE
Unbiased = False

4b. Multi-Head Causal Self-Attention

The attention mechanism allows each token to aggregate information from all previous tokens in the sequence. It computes three projections of the input — Queries (Q), Keys (K), and Values (V) — using learned linear transformations, then computes scaled dot-product attention:

Attention(Q, K, V) = softmax(Q Kᵀ / √d_k) · V

Multi-head mechanism: The 384-dimensional embedding is split into 6 independent heads of 64 dimensions each. Each head learns different attention patterns (e.g., one head might focus on adjacent tokens for local syntax, another on distant tokens for long-range semantics). The outputs of all 6 heads are concatenated and projected back to 384 dimensions via W_out.

Causal masking: An upper-triangular mask fills future positions with -inf before the softmax, ensuring token t can only attend to tokens 0 through t. This is what makes the model autoregressive — it generates one token at a time, left to right, without peeking ahead.

4c. Residual (Skip) Connections

The output of each sub-layer is added back to its input before being passed to the next sub-layer. This creates a “gradient highway” — during backpropagation, gradients can flow directly through the skip connection without being transformed, which is essential for training deep networks. Without residual connections, a 6-layer model would suffer from vanishing gradients and fail to train.

A dropout layer (p=0.1) is applied to the sub-layer output before addition, following the original Transformer design.

4d. Position-wise Feed-Forward Network

A two-layer MLP applied independently to each token position. The hidden dimension expands to 4× the embedding dimension (1,536), then projects back down. This expansion-contraction pattern gives the model capacity to learn complex transformations at each position. Research suggests this is where the model stores factual knowledge (Geva et al., 2021).

FFN(x) = W_2 · GELU(W_1 · x + b_1) + b_2

GELU activation (Hendrycks & Gimpel, 2016) is a smooth approximation of ReLU that weights inputs by their percentile under a Gaussian distribution. Unlike ReLU’s hard cutoff at zero, GELU allows small negative values to pass through, which improves gradient flow.

4e. KV Cache (Inference Optimization)

Without caching, autoregressive generation is wasteful: to generate token t, the model recomputes Keys and Values for all previous tokens 1 through t−1 from scratch. For a 200-token response, the first token’s K/V is recomputed 200 times. KV caching eliminates this redundancy.

Prefill phase: The full prompt is passed through the model in one forward pass. The K and V tensors from every attention layer are stored in a cache — a list of (K, V) tensor pairs, one per block.

Decode phase: For each new token, only that single token is passed through. Its Query attends to the cached Keys/Values from all previous tokens, then its own K/V is appended to the cache. The cost per token drops from O(seq_len × d_model) to O(d_model).

Without cache: model(all_tokens) × N steps
With cache: model(prompt) once, then model(new_token, cache) × N steps

The cache is threaded through the model: GPTModel.forward() accepts an optional kv_caches list and a start_pos integer for correct positional embedding. Each MultiHeadAttention layer concatenates new K/V tensors to its cached tensors and returns the updated cache.

5. Output Head

After passing through all 6 transformer blocks, a final LayerNorm is applied to stabilize the representations. Then a linear projection maps from the embedding dimension (384) to the full vocabulary size (50,257), producing a vector of logits — unnormalized log-probabilities over every possible next token.

During training, these logits are compared to the true next token using cross-entropy loss. During inference, we convert logits to probabilities and sample the next token:

Decoding Strategy

Temperature scaling (T=0.8): Divides logits by T before softmax. Lower temperatures sharpen the distribution (more deterministic), higher temperatures flatten it (more creative). We use 0.8 for a balance of coherence and variety.

Top-k filtering (k=40): Only the 40 highest-probability tokens are considered; the rest are masked to -inf. This prevents the model from sampling extremely unlikely tokens that would produce gibberish.

The combination: next_token = sample(softmax(top_k(logits / T)))

AdamW (lr=5e-4, wd=0.1)

4-Bit Quantization (NF4)

The base model weights are quantized from 16-bit to 4-bit using the NormalFloat4 (NF4) data type — optimal for normally distributed neural network weights. Double quantization further compresses the quantization constants themselves, reducing memory by ~75% with minimal quality loss.

LoRA Adapters

Instead of updating all 1.5B parameters, LoRA freezes the base model and injects small trainable rank-decomposition matrices into every attention and MLP layer. Only ~0.5% of parameters are trained, making fine-tuning fast and memory-efficient.

Philosophy
What is Stoicism?

Stoicism is a school of Hellenistic philosophy founded in Athens by Zeno of Citium around 300 BCE. It teaches that virtue — the highest good — is based on knowledge, and that the wise live in harmony with the divine reason that governs nature. Destructive emotions result from errors in judgment, and a sage would not suffer from such emotions.

“The happiness of your life depends upon the quality of your thoughts: therefore, guard accordingly.” — Marcus Aurelius, Meditations
The Four Virtues
Pillars of Stoic Ethics
  • I
    Wisdom (Sophia) — The ability to navigate complex situations with informed judgment and calm deliberation. Not mere cleverness, but deep understanding of what truly matters.
  • II
    Courage (Andreia) — Not just physical bravery, but moral courage: the strength to act rightly in the face of fear, pain, or social pressure. To speak truth when silence would be easier.
  • III
    Justice (Dikaiosyne) — Treating others with fairness, kindness, and good faith. The Stoics believed this was the most important virtue, as it governs our relations with others.
  • IV
    Temperance (Sophrosyne) — Self-discipline and moderation in all things. The mastery of desire, the governance of appetite, and the practice of restraint.

The Philosophers
Voices in the Corpus

Marcus Aurelius

121–180 CE · Roman Emperor

The philosopher-king. His Meditations were private journals never intended for publication — raw, unpolished reflections on duty, mortality, and the discipline of perception. He wrote them in Greek during military campaigns along the Danube frontier.

Seneca

c. 4 BCE–65 CE · Statesman & Tutor

Advisor to Emperor Nero and one of Rome’s wealthiest men, Seneca wrote with literary elegance about the shortness of life, the management of anger, and the path to tranquility. His Letters to Lucilius are a masterclass in practical philosophy.

Epictetus

c. 50–135 CE · Former Slave

Born into slavery, Epictetus earned his freedom and became one of the most influential Stoic teachers. His Discourses and Enchiridion focus on the dichotomy of control: distinguishing what is up to us from what is not.

Core Teachings
Key Stoic Principles
  • I
    The Dichotomy of Control — Focus only on what you can control (your thoughts, actions, responses) and accept what you cannot (others’ behavior, external events, the past).
  • II
    Amor Fati — Love of fate. Not merely accepting what happens, but embracing it as necessary and good. Every obstacle is material for virtue to work on.
  • III
    Memento Mori — Remember that you will die. This is not morbid pessimism but a call to urgency: live fully now, waste nothing, let no day pass without purpose.
  • IV
    Premeditatio Malorum — The premeditation of adversity. Visualize challenges before they arrive so they cannot ambush you. The prepared mind is the resilient mind.
  • V
    Sympatheia — The interconnectedness of all things. We are parts of a larger whole, and our actions ripple outward. What injures the hive injures the bee.
“It is not things that disturb us, but our judgments about things.” — Epictetus, Enchiridion
